1. Context

1-1. 問題定義

既存 /publish skill は md → HTML テンプレ機械適用のみで、アートディレクター監修不在。出力は「Inter フォント + 紫グラデーション + 機械的箇条書き」型の AI スロップに陥り、ジョニー基準で質低判定。

1-2. 根本原因 (ai-content-ops 分析 3 点)

  • A. publish が designer / frontend-design と断絶 (テンプレが出力の天井を決める構造)
  • B. 「図表多め / 図解ビジネス書」等の修飾語が Pipeline 上で意味論的に消滅 (スタイル判定が YAML frontmatter ベース)
  • C. designer skill の「Web UI 実装時は frontend-design 併用」「自己検証モード」が publish 経由では発動しない (二重断絶)

1-3. 設計哲学

  • 3 段ゲート構造で全デザイン skill 横断統一 (publish / gdocs / marp / svg / 動画 / 印刷)
  • デフォルト = subagent 内部完結 (PASS で自動公開)、--review フラグで G5 Discord 承認切替
  • コンセプト判定は素子自動推定、不明時のみ 2 択 (user-autonomy-trust.md 準拠)
  • sanity-check-protocol.md の「判定責任を subagent に委譲」設計思想を踏襲

1-4. ジョニー確定要件 (本セッション AskUser 回答)

  • スコープ: 全デザイン skill フル展開
  • 監修パターン: subagent 内部完結デフォルト + --review で G5 切替
  • コンセプト判定: 素子自動推定 + 不明時 2 択
  • 既存 publish-skill-twostep: G3 観察済 archive、本 Plan に統合

2-3. verdict JSON contract

{
  "verdict": "PASS" | "MODIFY_REQUIRED" | "REJECT",
  "score": {
    "typography": 0-10,
    "color": 0-10,
    "hierarchy": 0-10,
    "visual_weight": 0-10,
    "grid": 0-10,
    "consistency": 0-10,
    "medium_fit": 0-10,
    "brand_alignment": 0-10
  },
  "issues": [
    { "axis": "typography", "severity": "high|med|low",
      "description": "...", "fix": "..." }
  ],
  "revision_prompt": "再生成用具体的指示 (MODIFY_REQUIRED 時のみ)",
  "references": ["masters-analysis.md §X", "media/slide.md §Y"]
}

verdict 閾値:

  • PASS: score 平均 ≥ 7.5 かつ high severity = 0
  • MODIFY_REQUIRED: high 1-2 件 または平均 5.5-7.5
  • REJECT: high ≥ 3 件 または平均 < 5.5

2-4. AI スロップ自動検出 (art-director 監修の Step 1、機械チェック)

ai-content-ops の AI スロップ回避基準 5 項目を art-director 監修の冒頭機械チェックで適用:

# チェック 失格条件
A 禁止フォント検出 (CSS grep) Inter / Roboto / Arial / Space Grotesk が font-family に含まれる
B スロップ配色検出 (CSS grep) 紫グラデ (hsl 240-300 + linear-gradient) + 白背景 / pastel blue 大量
C 情報密度均一性 連続箇条書き ≥ 7 行で余白セクションなし / H2 間視覚区切りなし / 全テキスト無ビジュアル
D コントラスト比 (Python colorsys) テキスト × 背景 < 4.5:1 (WCAG AA)
E 概念ビジュアル ビジネス書要求時、H2 セクションに SVG/図解/データ可視化のいずれかが 0

1 件以上ヒット = high severity 自動付与。

2-5. ジョニー向けデフォルトトーン (Axis 2 = 不要時の brand=neutral 定義)

ai-content-ops 言語化 7 項目を design-pipeline.md SSoT に記載:

  1. 低体温・無修飾 (「革新的」「圧倒的」等の装飾語禁止)
  2. 体感ベース叙述 (統計より「手触り」「場の空気」「時間感覚」)
  3. 押しつけない結論 (「〜べき」禁止、余白を残す)
  4. 文章プロ視点 (120-150 字段落、リズム変化)
  5. 淡々とした緊張感 (大文字・感嘆符・過剰絵文字なし)
  6. 図解は証拠 (キャプションは図の解説ではなく含意)
  7. 終わり方は開く (結論で締めない、問いか次の視点)

2-6. 媒体別品質チェックリスト (art-director が medium パラメタで分岐参照)

medium チェック項目 (5 件)
gdocs Noto Sans JP / 強調=太字+下線+黄マーカー / 表 padding ≥ 8px / ASCII 罫線禁止 / ページあたりビジュアル ≥ 1
HTML/web-page Noto Sans JP fallback / コントラスト ≥ 4.5:1 / H2 単位ビジュアル ≥ 1 / 3 ブレークポイント / スロップ配色 0
画像/banner 中心モチーフ 1 / レッドリスト色系 or 固有色 / テキストフォント ≥ 48px / AI 破綻 (手・歯・鏡・文字) 人間 gate / SNS 1:1 + note 1.91:1 セット
動画/video テロップ 40px+ / 静止 ≤ 3s / 開始 3s 内フック / 無音視聴可 / 出力 1920×1080 or 1080×1920

2-7. infinite loop 防止

  • art-director frontmatter disallowedTools: Agent で自呼び出し物理ブロック
  • revision 上限 2 回 (Open Q 1 で確定)、超過は REJECT 扱い
  • maxTurns: 3 で subagent 自身のターン数も上限
